# Bench Guides

Bench Guides should feel like a pre-shift notebook, not a technical database screen.

The goal is to give staff a quick read before starting a bench:

- What should I check first?
- What routine items are easy to forget?
- Which organism groups need current approved guidance?
- What needs handoff before I leave?

Because the current site is public GitHub Pages, detailed SOP steps, Epic build strings, organism-specific finalization instructions, patient examples, and code-level workup reminders should stay out of the live public app until OMC Micro has authentication or approved internal hosting.

## Public Version

The public version can safely show:

- Plain-language reminders
- Non-patient shift prep notes
- Organism group headings
- Senior-review prompts
- Legacy LIS explanation at a high level
- Private-detail placeholders

## Private Version Later

The private version can later support:

- Exact Epic workflow names
- Code-level reminders
- Approved SOP report phrases
- Organism-specific workup details
- Reviewed local bench tips
- Versioning, owner, review date, and audit trail

## Content Rule

Use this rule when adding notes:

```text
If it sounds like "remember to check this before shift," it can probably be public-safe.
If it sounds like exact procedure, exact Epic steps, exact report wording, or exact organism workup, keep it private.
```
